02 7-Layer Architecture

Click any layer to expand it and see which repos power it, how they connect to Chem Connect, and where each one plugs into the existing codebase.

How to read this: Each layer builds on the one below it. Layer 1 (AI Foundation) powers everything. Layers 3–4 feed data into Layer 5 (CRM). Layers 5–6 drive Layer 7 (Content Engine). The result is a fully autonomous loop: discover leads → enrich → score → generate content → send → track → follow up.

04 3-Phase Build Roadmap

Sequenced so each phase delivers immediate value while building toward the full AI engine. Phase 1 can ship in days, not weeks.

Phase 1 · Week 1–2
AI Email Foundation
Immediate value. Builds on existing Mailgun integration.
  • Install Vercel AI SDK + @ai-sdk/anthropic. Wire Claude to a new /api/ai/generate route.
  • Install official Mailgun MCP server. Claude can now send emails natively via agent.
  • Install Trigger.dev. Wrap AI generation in background jobs (no Vercel timeout limits).
  • Build /admin/marketing page with campaign builder. Paste or generate HTML email, select contacts from existing profiles table, send.
  • Wire CoppieGPT frameworks into Claude prompt. Generate 5 email variants (AIDA, PAS, 9-Word, OCA, BAB) — you pick one. Store in content_variants table.
  • Add contacts table migration. Import existing profiles as first contacts.
  • Basic campaign analytics: open rate, click rate via Mailgun webhooks.
Phase 2 · Month 1
Lead Generation Engine
Build the prospect database. All of Australia as your market.
  • Set up Google Maps Scraper. Run searches: "concrete batching plant QLD/NSW/VIC/WA/TAS/SA", "quarry [state]", "ready mix concrete [city]". Feed results to contacts table.
  • Integrate ABN Lookup API. Validate every scraped business. Filter for active, GST-registered companies only. Append official registered name.
  • Install Apollo.io MCP. Enrich contacts with decision-maker names, LinkedIn, direct emails, job titles.
  • Install Hunter.io MCP. Find verified email addresses for procurement managers at target companies.
  • Build n8n pipeline: Maps scrape → ABN validate → Apollo enrich → score → auto-import to Supabase contacts table.
  • Add Listmonk for SQL-based segmentation. Segment by state, industry, order recency, company size.
  • Install Twilio MCP. Add SMS campaign capability alongside email.
Phase 3 · Month 2–3
Full Autonomous Engine
Autonomous loop. Agents research, draft, you approve, system executes.
  • Install Mastra. Build multi-agent workflows: research-agent → scoring-agent → content-agent → human-approval → send-agent.
  • Adapt ai-sales-team-claude skills as Mastra agent tools. ICP builder trained on Heidelberg/Boral/Holcim customer profile.
  • Set up Inngest event triggers: order.lapsed_30_days → re-engagement sequence; email.opened → Day 3 follow-up; new_lead.high_score → Jonny alert; new_order → reward email.
  • Build visual drip campaigns using Mautic or Laudspeaker. Day 1 intro → Day 3 case study → Day 5 pricing → Day 14 SMS → Day 21 call prompt.
  • Implement Deal Resurrector from ai-marketing-skills. Track departed contacts to new companies.
  • Supabase pgvector + RAG over all 23 Chem Connect product specs, MSDS sheets, pricing. Claude generates technically accurate content with no hallucinated specs.
  • Deploy GHL MCP if using GoHighLevel. 520+ tools give Claude complete CRM control.

Key Behaviours & UX Patterns

🤖
Human-in-the-Loop, Always
Claude never sends anything autonomously. Every campaign pauses at Step 4 for Jonny to review and approve. The AI generates options — humans make decisions. The Trigger.dev workflow literally suspends until you click "Send."
Trigger.dev: await io.waitForEvent("campaign.approved")
Mastra: workflow.suspend() → resume on approval
No campaign fires without explicit human confirmation
⚡
Existing Integrations, Extended
Marketing doesn't replace anything — it extends what's already built. The existing sendEmail() in lib/email/send.ts is called by the campaign bulk-sender. Stripe webhooks trigger Inngest automations. Xero's xero_contact_id links to CRM contacts.
lib/email/send.ts → extended with sendBulkCampaign()
app/api/webhooks/stripe → fires inngest events
profiles.xero_contact_id → links to contacts table
🎯
Segment-Aware Content Generation
Claude knows the audience when generating content. The prompt is automatically enriched with segment data (NSW quarry operators, concrete plants, fleet operators) and product RAG context (specs, pricing, MSDS). Content is never generic — it's targeted to the specific audience selected in Step 1.
Context injected: segment tags + product embeddings
RAG: supabase pgvector over all 23 product specs
Personalisation: {company}, {contact_name}, {state}
📊
Behaviour-Triggered Automations
Inngest listens to real database events from Supabase. When an order is marked "delivered" → rewards email fires. When 30 days pass with no order → re-engagement SMS queues. When a contact opens an email → Day 3 follow-up is scheduled. All without any manual action.
order.delivered → rewards/upsell sequence
customer.lapsed_30d → re-engagement SMS via Twilio
email.opened → follow-up drip triggered
new_lead.scored → alert notification
