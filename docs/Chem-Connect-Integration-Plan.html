How It Works (4-Step Flow)

1
Customer Enters Postcode at Checkout
Our system calls the MacShip Get Routes API to get live carrier pricing.
POST https://live.machship.com/apiv2/routes/returnRoutes

We send: pickup warehouse (closest to customer), delivery postcode, item details (packaging size, weight, dimensions), DG classification, tailgate requirement (based on forklift selection).

MacShip returns multiple carrier options with prices. We display the shipping cost to the customer.

2
Customer Places Order (Stripe or PO)
Our system calls the MacShip Create Consignment API to book the actual shipment.
POST https://live.machship.com/apiv2/consignments/createConsignment

We get back: MacShip consignment ID, carrier consignment ID, and tracking information. These are stored on the order in our database and visible in both admin and customer dashboards.

3
Admin Dispatches the Order
When you update the order status, we call the MacShip Manifest API to dispatch.
POST https://live.machship.com/apiv2/manifests/manifest

This notifies the carrier to pick up the goods from the warehouse. Shipping labels are auto-generated and can be printed directly from the admin dashboard.

4
Live Tracking
We periodically sync tracking updates from MacShip.
GET https://live.machship.com/apiv2/consignments/getConsignment?id={id}

Tracking status and Proof of Delivery (POD) are synced to our order tracking system. Customers see real-time updates on their dashboard, and you see them in admin.

Warehouse Selection Logic

When a customer places an order, the system automatically selects the best warehouse:

  1. Check which warehouses carry the ordered product (based on product's region setting)
  2. From available warehouses, pick the closest one to the customer's postcode
  3. Use that warehouse as the pickup location for the MacShip quote

Part 2: Xero Integration (Accounting)

A. Automatic Invoicing (for Purchase Order Customers)

Current Flow (Manual)

  1. Customer pays by Purchase Order on our platform
  2. Jonny manually goes to Xero, creates an invoice, adds the PO number
  3. Sends the invoice to the customer

New Flow (Automated)

  1. Customer pays by Purchase Order on our platform
  2. System automatically checks if customer exists as a Xero Contact (by email)
  3. If not, creates them as a new Xero Contact
  4. Creates an Invoice in Xero with:
    • Customer's PO number as reference
    • All line items (product name, qty, unit price)
    • Shipping cost (from MacShip)
    • GST (10%)
    • Due date (based on your standard payment terms)
  5. Invoice set to AUTHORISED status (ready to send)
  6. You can review in Xero and send, or we can auto-send
API Used:
PUT https://api.xero.com/api.xro/2.0/Invoices (Create Invoice, Type: ACCREC)
PUT https://api.xero.com/api.xro/2.0/Contacts (Create Contact if new customer)

B. Automatic Purchase Orders (to Warehouses)

Current Flow (Manual)

  1. Customer places order on our platform
  2. Jonny manually goes to Xero, creates a Purchase Order to the warehouse
  3. Warehouse receives PO and makes/prepares the product

New Flow (Automated)

  1. Customer places order on our platform
  2. System determines which warehouse to order from (based on product region + closest to customer)
  3. Creates a Purchase Order in Xero to that warehouse with:
    • Warehouse as the Xero Contact (supplier)
    • Product details (name, qty, packaging size)
    • Warehouse-specific pricing (each warehouse may charge differently)
    • Customer delivery address
    • Reference to our platform order number
  4. Purchase Order is sent to the warehouse via Xero
API Used:
PUT https://api.xero.com/api.xro/2.0/PurchaseOrders (Create PO)
Warehouse contacts are already set up in your Xero account - we fetch them via the Contacts API.

C. Customer Contact Sync

When a new customer signs up on Chem Connect, we automatically create them as a Contact in Xero. This means invoices can be issued immediately without manual setup.

Data we already collect at registration (no changes needed):

  • Company name, Contact name, Email, Phone, ABN, Business address
API Used:
PUT https://api.xero.com/api.xro/2.0/Contacts (Create Contact on signup)
GET https://api.xero.com/api.xro/2.0/Contacts (Check if contact exists before creating)

Xero Authentication

Xero uses OAuth 2.0. We'll build a "Connect to Xero" button in the admin dashboard. You click it once, authorize the connection, and it stays connected (we handle token refresh automatically).
