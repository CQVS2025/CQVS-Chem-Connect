Part B — Foundations (Week 1–2 must-haves)

2. Sitemap & robots.txt

What I’ll build

  • Dynamic sitemap at app/sitemap.ts — pulls live product slugs, category slugs, and content pages from Supabase. Updates automatically on every deploy and on revalidation.
  • Sitemap index split into logical sections so it scales when the catalog grows past a few hundred SKUs:
    • /sitemap.xml — index pointing to children below
    • /sitemap-static.xml — home, compliance, privacy, terms, contact, about, blog index
    • /sitemap-products.xml — one URL per product (auto-generated)
    • /sitemap-categories.xml — one URL per category landing page
    • /sitemap-locations.xml — one URL per state + per warehouse city (10 entries)
    • /sitemap-content.xml — knowledge hub articles, SDS pages, FAQs
    • /sitemap-images.xml — product photos with captions (Google Images discovery)
  • robots.txt at app/robots.ts:
    • Allow: /, /products/*, /categories/*, /chemical-supplier/*, /compliance, /privacy, /terms
    • Disallow: /cart, /checkout, /login, /register, /forgot-password, /reset-password, /dashboard/*, /admin/*, /api/*
    • Sitemap reference and host directive
  • Submit to Google Search Console + Bing Webmaster Tools and monitor coverage weekly.
What you get
  • Every product is discovered by Google automatically — no manual submission ever again.
  • Private pages (cart, checkout, account, admin) stay out of Google permanently.
  • Full visibility in Search Console of which pages are indexed and which aren’t.
  • Image sitemap surfaces product photos in Google Images — a B2B discovery channel competitors ignore.

6. JSON-LD schema strategy

There is currently zero structured data anywhere in the codebase. This is the single biggest visual upgrade we can give Google — rich results take up 2–3× the screen real estate of plain results.

Organization

Site-wide (root layout). Includes legal name (Concrete & Quarry Vending Systems Pty Ltd), ABN, logo URL, customer contact email, support phone, social profiles, and address of head office.

WebSite + SearchAction

Site-wide. Tells Google about the site’s search box so it can render a sitelinks search box for branded queries (“chem connect acetone”).

Product

On every /products/[slug] page. Includes name, SKU, brand (manufacturer), CAS as gtin alternative, image, description, hazard pictograms via hasMerchantReturnPolicy + custom properties.

Offer / AggregateOffer

Per pack-size (500 mL, 1 L, 5 L, 20 L drum, IBC). Pulls live AUD pricing + stock from Supabase. Includes priceCurrency: AUD, availability, itemCondition, and per-warehouse shippingDetails.

BreadcrumbList

Home → Category → Sub-category → Product. Renders as breadcrumb trail in search results, lifts CTR.

LocalBusiness × 5

One per active warehouse: ChemBuild Melbourne, Geelong site, Environex (QLD) Brisbane, QLD Shed Gold Coast, Chemology Adelaide, Environex (WA) Perth, NSW Shed Newcastle. With opening hours, geo coordinates, service area. Sydney coverage uses Service schema (no physical address there).

FAQPage

On /compliance (already a long-form Q&A page) and on every product page’s FAQ section (minimum order, lead time, dangerous-goods handling, bulk pricing).

Article / BlogPosting

On every knowledge-hub article. Includes author, datePublished, dateModified, publisher, and image.

AggregateRating + Review

Once reviews are collected (see Part G). Lights up star ratings in search results — widely shown to lift CTR by 25%+.

Sydney coverage strategy — even without a warehouse there

Decision point: Sydney has more B2B chemical buyers than any other Australian city. Walking away from those queries would be a strategic miss. Cardiff / Newcastle is ~140 km from Sydney CBD — a 1.5–2 hour run, well within next-day freight territory. We can target Sydney queries honestly without a Sydney warehouse, by being upfront about dispatch.
  • Build a Sydney coverage landing page at /chemical-supplier/sydney with content like:
    • “Chem Connect dispatches to Sydney metro within 1–2 business days from our Newcastle hub.”
    • Map showing the Newcastle dispatch point + Sydney delivery radius.

21.5 AI search & GenAI citation optimisation (2026 priority)

Why this is now critical: in 2025–2026 a meaningful share of B2B research moved out of classic Google search and into Google AI Overviews, ChatGPT, Perplexity, Microsoft Copilot, and Claude. These systems answer queries by citing trusted, well-structured web pages — getting cited becomes the new “ranking”. Most chemical sites haven’t adapted; this is a window we should walk through now.

What gets a page cited by AI search systems

  • Well-structured, fact-dense content — clear headings, definitions, tables, FAQs, comparison structures. The kind of content that’s easy for a model to extract.
  • Schema.org markup (already covered in Part C) — Product, FAQPage, HowTo, ChemicalSubstance, Article. AI systems lean on schema heavily for entity disambiguation.
  • Author bylines + dates on content. Models penalise undated, unattributed copy.
  • Llms.txt — an emerging convention (mirroring robots.txt) signalling which content is canonical / preferred for citation. Cheap to add, marginally future-proof.
  • Original data + opinions — pricing tables, freight lead times, comparison matrices, safety notes specific to AU regulations. Generic copy doesn’t get cited; specific, sourced facts do.
  • Brand attribution within the citation window — DONE AI systems usually cite a single sentence; if “Chem Connect” isn’t in that sentence, the citation goes to a competitor or to a generic source. The ensureBrandMention() utility in lib/seo/helpers.ts guarantees the brand name lands within the first 100 words of every auto-generated intro on programmatic pages (/brand, /buy, future /uses, /surfaces, etc.).
  • Crawl access for AI bots — GPTBot (OpenAI), PerplexityBot, ClaudeBot (Anthropic), Google-Extended. Decision needed: allow them to index the public catalog (recommended for visibility), or block them. Default to allow.

What I’ll build

  • app/llms.txt/route.ts serving an llms.txt file — lists canonical content URLs, summary of the business, and citation guidance for LLMs.
  • Updated robots.ts with explicit allow rules for GPTBot, PerplexityBot, ClaudeBot, Google-Extended. Disallow them from /admin, /api, etc. (same as Googlebot).
  • FAQ blocks on every product, category, location, and SDS page — structured Q&A with FAQPage schema. AI systems prefer FAQ-formatted answers.
  • Comparison tables on category pages (e.g. industrial-grade vs lab-grade acetone, drum vs IBC) — AI systems cite tables disproportionately often.
  • Author bylines on every knowledge-hub article: named author, role, link to a real bio page (/about/team).
  • Citation-friendly summaries at the top of long content pages (1–2 sentences, fact-dense) — the bit that gets quoted.
  • Test prompts monthly against ChatGPT, Perplexity, and Google AI Overviews for queries like “Australian industrial chemical supplier with bulk pricing”. Track citation rate as a KPI.
What you get
  • Citations in ChatGPT, Perplexity, Google AI Overviews, Copilot when buyers ask “where do I buy {chemical} in Australia?”.
  • Defensive position as classic search shifts toward AI summaries.
  • Same structural changes that help AI also help classic Google rankings — double dividend.
  • Trackable: citation count over time becomes a north-star KPI alongside organic sessions.

C3 — sitemap.ts

// app/sitemap.ts
import type { MetadataRoute } from "next"
const SITE = "https://www.cqvs-chemconnect.com.au"

async function getActiveProductSlugs() {
  const res = await fetch(
    `${process.env.NEXT_PUBLIC_SUPABASE_URL}/rest/v1/products?is_active=eq.true&select=slug,updated_at`,
    {
      headers: {
        apikey: process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
        Authorization: `Bearer ${process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!}`,
      },
      next: { revalidate: 600 },
    },
  )
  return res.ok ? res.json() : []
}

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const products = await getActiveProductSlugs()
  const now = new Date()
  const staticRoutes: MetadataRoute.Sitemap = [
    { url: `${SITE}/`, lastModified: now, changeFrequency: "weekly", priority: 1.0 },
    { url: `${SITE}/products`, lastModified: now, changeFrequency: "daily", priority: 0.9 },
    { url: `${SITE}/rewards`, lastModified: now, changeFrequency: "monthly", priority: 0.6 },
    { url: `${SITE}/compliance`, lastModified: now, changeFrequency: "yearly", priority: 0.4 },
    { url: `${SITE}/privacy`, lastModified: now, changeFrequency: "yearly", priority: 0.3 },
    { url: `${SITE}/terms`, lastModified: now, changeFrequency: "yearly", priority: 0.3 },
  ]
  const productRoutes: MetadataRoute.Sitemap = products.map(
    (p: { slug: string; updated_at: string }) => ({
      url: `${SITE}/products/${p.slug}`,
      lastModified: new Date(p.updated_at),
      changeFrequency: "weekly" as const,
      priority: 0.8,
    }),
  )
  return [...staticRoutes, ...productRoutes]
}

Note: as the catalog grows past a few hundred SKUs, this gets split into sitemap-products.xml, sitemap-categories.xml, etc. (see Section 2). Keep it simple for now.
