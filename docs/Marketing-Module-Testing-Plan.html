1.4 — Register GHL workflow webhooks (contacts + inbound SMS only)

Two workflows now, inside the CQVS sub-account → Automation → Workflows. The former "Email Events" workflow is deprecated — email attribution moved to the Marketplace App LCEmailStats webhook (next section). If the old CC — Email Events workflow still exists, keep it in Draft so it can't double-count against LCEmailStats.

Both workflows use the same custom header on their webhook action:

x-ghl-signature: <value of GHL_WEBHOOK_SECRET>

Pick your destination URL:

  • Local: start ngrok (ngrok http 3000) → copy the HTTPS URL. Replace <BASE_URL> below with it.
  • Production: e.g. https://www.cqvs-chemconnect.com.au.

Workflow 1 — Contact Sync

  1. + Create Workflow → Start from Scratch. Name: CC — Contact Sync.
  2. Add triggers: Contact Created, Contact Changed, Contact DND, Contact Tag, Contact Deleted.
  3. Add action Webhook:
    • Method: POST
    • URL: <BASE_URL>/api/webhooks/ghl/contacts
    • Custom Headers: x-ghl-signature = your secret
    • Custom Payload (JSON): 
      {
  "type": "{{trigger.name}}",
  "contactId": "{{contact.id}}",
  "locationId": "{{location.id}}",
  "email": "{{contact.email}}",
  "phone": "{{contact.phone}}",
  "firstName": "{{contact.first_name}}",
  "lastName": "{{contact.last_name}}",
  "tags": "{{contact.tags}}"
}
  4. Publish.

Workflow 2 — Inbound SMS

  1. + Create Workflow. Name: CC — Inbound SMS.
  2. Trigger: Customer Replied. Filter: Reply Channel = SMS.
  3. Add Webhook action:
    • URL: <BASE_URL>/api/webhooks/ghl/conversations
    • Custom header: x-ghl-signature = your secret
  4. Publish.
Do NOT publish a workflow-based CC — Email Events webhook in rev 3.
The LCEmailStats Marketplace webhook (section 1.5) now handles all opens / clicks / bounces / unsubscribes / complaints with deterministic per-message attribution. Running both paths in parallel causes double-counting on every open and every click.

1.5 — GHL Marketplace App (LCEmailStats) NEW

This is the big change in rev 3 — it fixes the attribution bug where two campaigns sent seconds apart showed all opens/clicks on only one of them. The workflow-based "Email Events" trigger does not include the per-email message ID, so attribution degraded to "most recent anchor for this recipient." The Marketplace App's LCEmailStats webhook carries lc-operations.email_message_id on every event, which matches exactly what the dispatcher stores at send time.
